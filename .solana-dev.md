# Solana Development Rules for NeuroSwarm AI

## Stack constraints (enforced)

1. **Dashboard (React/Next.js)**
   - Import RPC via `useSolana()` from `@/context/SolanaContext`
   - Import wallet via `useWallet()` from `@/context/WalletContext`
   - Use `Address` type from `@solana/kit`, NEVER `PublicKey` from `@solana/web3.js`
   - All web3.js usage MUST go through `dashboard/src/lib/solana/web3-compat-adapter.ts`

2. **On-chain program (Rust)**
   - Anchor 0.29.0 is the default framework
   - NO floating-point arithmetic — use integer math for all calculations
   - Use `solana_program::pubkey!()` for compile-time program ID constants
   - All PDA seeds defined in `constants.rs`
   - Bounds-check all Vec operations

3. **Testing**
   - Rust unit tests: LiteSVM (fast, in-process)
   - Instruction-level tests: Mollusk
   - Integration tests: Surfpool (surfpool.toml)
   - Python tests: pytest + pytest-asyncio

4. **Client generation**
   - Run `anchor build` to generate IDL
   - Run Codama (`codama.config.mjs`) to generate typed TS clients

5. **Security**
   - Never commit keypairs — `keys/` is in `.gitignore`
   - RPC endpoints via env vars (`NEXT_PUBLIC_RPC_URL`)
   - Always specify commitment level explicitly
   - Add compute budget instructions for mainnet transactions
